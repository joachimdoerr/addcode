h2. Addcode Addon für Redaxo

Dieses Addon ermöglicht es eigene Functions und Classes laden zu lassen. Dabei berücksichtigt das Addon anhand des Dateinamens ob der jeweilige Code für das Front- oder Back-End geladen wird. Es können Dateien auch global geladen werden. Das Addon ermöglicht es auch aus einem beliebigen Ordner her aus Functions oder Classes zu laden. 

h3. Dateinamen

Die Dateinamen müssen nach folgendem Schema aufgebaut sein "typ.name.ladeziel.inc.php" (typ = "classe" oder "function", name = Name der Datei -> beliebig, ladeziel = "frontend", "backend" oder "global").

h5. Folgende Dateinamen wären also zulässig:

* class.customnavigation.frontend.inc.php
* class.customstruktur.backend.inc.php
* class.taggin.global.inc.php
* function.getimagestoarray.frontend.inc.php
* function.getpreviewimage.backend.inc.php
* function.getfiletype.global.inc.php


h5. Folgende Dateinamen würden ignoriert:

* classes.navigation.php
* function.splitit.inc.php
* functions.inc.php

h3. Laden von Functions

Functions die geladen werden sollen müssen im Verzeichnis "/redaxo/include/addons/addcode/include/functions/" abgelegt werden und wie bereits beschriebenen benannt sein.

h3. Laden von Classes

Classes die geladen werden sollen müssen im Verzeichnis "/redaxo/include/addons/addcode/include/classes/" abgelegt werden und wie bereits beschriebenen benannt sein.

h3. Custom Verzeichnis

Es kann zusätzlich ein Verzeichnis gewählt werden aus welchem Functions und Classes geladen werden, dieses Verzeichnis kann ein Redaxouser mit entsprechenden Berechtigungen unter "Addcode" Settings definieren, das Verzeichnis wird vom Dokument-Root aus gesetzt, dabei muss immer nach dem Ordnernamen ein "/" gesetzt werden. Der Ordner "site/include/" ist vordefiniert. Es werden nur dann Dateien geladen wenn sich in dem Custom Verzeichnis die Unterverzeichnisse "functions/" und "classes/" befinden. Die darin befindlichen Dateien werden dann daraus geladen wenn sie wie bereits beschriebenen bennant sind.
